THiS CRM v0.13.33 - dependency cache repair and automatic Mailchimp removal

WHY THE PREVIOUS DEPLOY FAILED
The Netlify log shows npm restoring or finding an old node_modules tree and then failing while renaming:
  node_modules/bare-stream -> node_modules/.bare-stream-*
with ENOTEMPTY. This is a stale/corrupted dependency tree problem, not a CRM compile error and not a Mailchimp error.

DEPLOYMENT CHANGES
- Automatic Mailchimp API syncing has been removed from the intake Netlify Function.
- Mailchimp environment variables and setup documentation have been removed.
- The manual Excel and Mailchimp-ready consent CSV exports remain available.
- Netlify now uses Yarn Classic 1.22.22 instead of npm for dependency installation.
- package-lock.json and npm-only configuration have been removed to avoid package-manager ambiguity.
- .gitignore now explicitly excludes node_modules and Netlify local-state folders.

FIRST DEPLOYMENT
Use Netlify's Clear cache and deploy site option for this release. The deploy log should say that packages are being installed using Yarn rather than npm.
