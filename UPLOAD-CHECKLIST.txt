THiS CRM v0.13.28 — Public Form Standardisation

Before deploying:
[ ] Preserve the existing Netlify Database and Microsoft Graph environment variables.
[ ] Create or identify the Mailchimp audience that should receive intake contacts.
[ ] Create a restricted Mailchimp Marketing API key for this integration.
[ ] Confirm the audience has FNAME and LNAME merge fields, or adjust the merge-field environment variables.
[ ] Add the Mailchimp environment variables from .env.example in Netlify.
[ ] Keep MAILCHIMP_DOUBLE_OPT_IN=true unless there is a documented reason to use single opt-in.
[ ] No database migration is required.

After deploying:
[ ] Submit a test intake without marketing consent.
[ ] Confirm the intake appears in THiS CRM and the Mailchimp contact is transactional/non-subscribed.
[ ] Confirm tags include Guided Intake, Website Assessment and the selected goal.
[ ] Submit a second test intake with marketing consent.
[ ] Confirm the Mailchimp contact is pending and receives the double opt-in email.
[ ] Complete the Mailchimp confirmation and verify the contact becomes subscribed.
[ ] Confirm health, character, immigration-history answers and CV uploads do not appear in Mailchimp.
[ ] Temporarily use an invalid API key and confirm the intake still saves and adviser notifications still send.
[ ] Restore the valid API key and redeploy.
